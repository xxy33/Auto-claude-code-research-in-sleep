# 启动速查（个人用）

> 简明记录：什么情况用什么命令。在项目根目录开 `claude` 后直接输入斜杠命令即可。
> 标 🆕 的是本次新增的「交给同事做实验」工作流（实现完成后可用）。

## 0. 前提

- Codex MCP 已连上（`claude mcp list` 里 `codex ✓ Connected`）→ 跨模型评审/校验才会生效。
- 每个研究方向单独一个文件夹：`research-projects/<slug>/`，产物都在里面，互不干扰。

## 1. 🆕 规划 → 交给同事做实验 → 收回结果写论文

| 我的情况 | 命令 |
|---|---|
| 已经有方法/idea，想直接出「实验交接单」 | `/experiment-handoff "方法或idea的一句话描述"` |
| 只有大方向，想先找idea再规划 | `/experiment-handoff "研究大方向" — discover: true` |
| 想自己指定项目文件夹名 | 加 `— slug: 我的项目名` |
| 同事把结果填进交接单返回后，开始写论文 | `/handoff-intake "research-projects/<slug>/EXPERIMENT_HANDOFF.md"` |
| 结果还没填全也想先动笔 | 加 `— allow-partial: true` |

产物：`research-projects/<slug>/EXPERIMENT_HANDOFF.md`（发给同事的交接单）
→ 同事**只填表格里 ⬜ 的 `Actual` 列**（标 `REUSE` 的是从已有论文引用的、不用重跑）
→ 返回后 `/handoff-intake` 自动校验 + 写作 + 润色，论文写进 `research-projects/<slug>/paper/`。

> 关键点：交接单里 `Expected (pred)` 是**我们预测的数**，只用来对照，**绝不会**当成真实结果写进论文。

## 2. 现在就能用的单步命令（按需单独调用）

| 我想做的事 | 命令 |
|---|---|
| 从大方向找 idea（全流程） | `/idea-discovery "具体方向"` |
| 细化/打磨一个方案 | `/research-refine "粗略想法"` |
| 单独做实验规划 | `/experiment-plan` |
| 查新（有没有人做过） | `/novelty-check "idea"` |
| 找相关工作/文献综述 | `/research-lit "主题"` |
| 让 GPT 评审我的研究 | `/research-review` |
| 从结果写论文 | `/paper-writing "NARRATIVE_REPORT.md"` |
| 编译 LaTeX 出 PDF | `/paper-compile` |

## 3. 备注

- 跑实验**不归这个项目管**——本项目只负责「调研 + 规划 + 写作润色」，实验交给同事。
- 服务器/GPU 信息在 `CLAUDE.md`（只有需要本地跑点小东西时才用得上）。
